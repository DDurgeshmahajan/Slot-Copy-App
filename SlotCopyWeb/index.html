User Guide

Three Shortcuts.
Infinite Workflow.

SlotCopy uses timing-based logic gates — no complex chords, no mode switching. Your existing habits stay completely intact.

01

Save to a Slot

Ctrl+C then within 500ms 1–9

Select any text, press Ctrl+C as usual. Within half a second, press a number key. An on-screen flash confirms the save. Miss the 500ms window? The number key behaves normally — no accidents.

02

Paste from a Slot

V hold + 1–9

Hold V (don't release), then press the slot number. The text appears instantly in any app. The "V-hold gate" is genius in its simplicity — it's a chord only you know.

03

Standard Paste Still Works

Ctrl+V tap quickly (<250ms)

A fast tap of Ctrl+V passes right through to the OS as a normal paste. SlotCopy intelligently distinguishes a "tap" from a "hold" — completely transparent to your muscle memory.

Architecture

Built Like a System Utility.
Not Just an App.

Four precision-engineered layers operating at the Kernel-to-User boundary. This is why it's fast.

🖥️
Layer 1 · Win32 OS Bridge

Low-Level Keyboard Hook WH_KEYBOARD_LL

A system-wide hook installed via SetWindowsHookEx(13) intercepts every keystroke system-wide, before it reaches any application. Irrelevant keys skip through via CallNextHookEx in nanoseconds — zero systemic lag.

🧠
Layer 2 · Logic Brain

Timing-Based State Machine

Two precision Stopwatch timers gate every action: a 500ms copy window for slot assignment, and a 150ms V-hold gate to distinguish paste intent. V-key is swallowed at kernel level and either re-fired as simulated paste or routed to slot paste.

🔄
Layer 3 · Clipboard Sync & STA Marshalling

Thread-Safe Clipboard Operations

Windows Clipboard requires Single Threaded Apartment (STA) access. Hook thread dispatches via RunInSTA. Paste executes the Snapshot Pattern: save clipboard → inject slot → simulate Ctrl+V → wait 75ms → restore original. Your clipboard history is never corrupted.

💾
Layer 4 · Persistence & OSD Feedback

Smart Disk I/O + On-Screen Display

Slots stored as raw strings (not DataObject) — bypassing OLE overhead for instant R/W. System.Text.Json serializes to AppData\SlotCopyApp\slots.json only on state change. OSD feedback marshalled to UI thread via SynchronizationContext — zero cross-thread exceptions.
